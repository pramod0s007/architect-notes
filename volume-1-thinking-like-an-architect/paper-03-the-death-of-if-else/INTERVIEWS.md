# Interview Discussion

## Question

When should if-else be replaced?

## Wrong Answer

Always.

## Better Answer

Only when the conditional is experiencing pressure and is likely to continue growing.

## Questions To Ask

- What is changing?
- Why is it changing?
- Which bucket does it belong to?
- Is the complexity accidental or essential?
- Which abstraction best relieves the pressure?

## Key Insight

if-else is not a code smell.

Uncontrolled growth is.
